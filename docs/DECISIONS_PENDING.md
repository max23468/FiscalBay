# Decisioni Aperte

Decisioni ancora aperte che influenzano le prossime fasi.

## Fase 3

- dove raccogliere o esporre le metriche operative
- quale soglia minima usare per alert su bot fermo o errori consecutivi
- se aggiungere correlation id espliciti per polling, ordini e chat

## Fase 4

- confermare SQLite o passare a Postgres prima della beta privata
- disegnare il modello tenant-aware definitivo
- decidere il livello di audit log necessario per token e collegamenti account

## Fase 5

- scegliere il formato del callback server OAuth
- decidere dove ospitare la parte web di onboarding
- definire il flusso UX minimo tra Telegram, pagina web e callback finale

## Fase 6

- definire retention log
- definire retention token e dati ordine
- definire limiti operativi del servizio pubblico
